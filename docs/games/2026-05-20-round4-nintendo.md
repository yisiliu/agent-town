## 四、本局暴露的核心问题

### 🚨 问题 1：预言家幻觉了一个不存在的查验
**事实**：林可在 t=3 的 night-seer 阶段 ABSTAIN——LLM 输出为空，没有做任何 peek。
**幻觉**：在 t=38 遗言里，林可声称"首夜我查验了法尔肯——他是狼"，凭空编造。
**后果**：村民相信假信息，第二天把无辜的法尔肯推出去；同时让狼人马利欧"顺水推舟"。

**根本原因**：LLM 在 last-words prompt 里只知道自己的"身份"（预言家），但不知道自己在游戏里**实际做过什么**。模型为了符合身份设定，编造了 fitting 的历史。

**已修复**（commit `5b26dae`）：
- 添加 `groundingFacts()` 函数，把演员的真实游戏历史强制注入到每个 prompt
- 预言家无查验时：`【你的查验记录】**你目前尚未做过任何查验**...**不要编造查验结果**`
- 预言家有查验时：列出具体的查验对象 + 阵营，并加注"只能引用以上事实"
- 女巫：明确显示解药/毒药状态（已用/未用）
- 注入位置：白天发言、投票、警长归票、**遗言（最关键）**

### 🚨 问题 2：警长投票看不到候选人发言
**事实**：警下投票时，prompt 只显示候选人 ID + 名字 + publicLog。candidate speeches 在 visibleTurns 里有，但**完全没渲染**给投票者。
**后果**：村民只能凭名字感觉投票，没办法分辨马利欧的狼伪装。

**已修复**（commit `5b26dae`）：
- sheriff-vote prompt 现在显示每个候选人的完整 sheriff-claim 发言文本
- 加入了警长投票启发：「谁跳了神职？跳得是否可信？谁说得过于空泛？谁的人设和发言风格冲突？」
- 同样修复了 sheriff-pk-vote（显示 round-1 + PK 两轮发言）
